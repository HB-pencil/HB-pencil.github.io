Nov 22, 2019 | 计算机基础Android知识

Kotlin

  • 兼容性：Kotlin 与 JDK 完全兼容，保障了 Kotlin 应用程序可以在较旧的 Android 设备上运行而无任何问题。Kotlin 工具在 Android Studio 中会完全支持，并且兼容 Android 构建系统。
  • 性能：由于非常相似的字节码结构，Kotlin 应用程序的运行速度与 Java 类似。 随着 Kotlin 对内联函数的支持，使用 lambda 表达式的代码通常比用 Java 写的代码运行得更快。
  • 互操作性：Kotlin 可与 Java 进行 100％ 的互操作，允许在 Kotlin 应用程序中使用所有现有的 Android 库 。

异常

  • kotlin的所有异常都是UnCheckedException，不强制要求我们去捕获异常
  • 但是不捕获的异常依然会出现，所以建议还是做好异常检测

由于编译器不会去检查java代码，所以如果kotlin调用java的代码，而这个代码存在null的可能性，也可以编译通过并运行。当发生null，依然会出现问题，之前遇到到这样的坑。

协程

如何理解进程、线程和协程

我的一些理解：

我们知道早期的OS是单道程序运行的也不存在所谓线程，为了加快并发效率，才出现了多道程序运行的机制，也就是多进程OS。一切看起来似乎很完美，但是此时进程作为OS资源调度和CPU调度的基本单位，进程的创建、上下文切换和销毁开销都是巨大的，在多道程序并发的时候对于运行效率仍然是不够优秀的。能不能在保证并发的时候又能进一步提高效率呢？这时候线程就出现了，线程存在于进程中，线程可以共享进程从OS所获得的资源，同时线程作为CPU调度的基本单位而不再是进程，就可以保持原来的并发的同时又提高运行效率，因为显然线程的切换开销要比进程低多了。

协程的概念，据我了解，大多数编程语言，协程都是依赖于线程而存在，就像线程被称为”轻量级进程“一样，协程也被称为”轻量级线程“。不同的是，协程本质还是线程，而线程本质上还是由OS决定的，协程是为了使线程最大限度的利用CPU时间而出现的，同时也可以减少线程的创建。

Kotlin的协程也是类似的，底层是通过状态机去实现协程的挂起和恢复

协程完全通过编译技术实现(不需要来自 VM 或 OS 端的支持)，挂起通过代码来生效。基本上，每个挂起函数都转换为状态机，其中的状态对应于挂起调用。刚好在挂起前，下一状态与相关局部变量等一起存储在编译器生成的类的字段中。在恢复该协程时，恢复局部变量并且状态机从刚好挂起之后的状态进行。

使用协程的好处：

协程可以被挂起而不阻塞线程，线程的阻塞代价是昂贵的。尤其是创建线程数量是有限的，当线程被阻塞，那些重要的任务就有可能被延迟。协程的挂起几乎是没有代价的，它不需要切换上下文。协程的本质是最大限度的利用线程所获得的cpu时间，提高线程的效率。

启动协程的几种常见方式：

  • runBlocking{} //会使当前线程阻塞，直到协程执行完毕

  • GlobalScope.launch{} //不阻塞线程，返回一个Job，可以取消协程

  • GlobalScope.async{}

    //不阻塞线程，返回一个Job，可以取消协程，可以通过await获取协程返回值，用法类似dart的async，只不过dart单线程通过微任务队列（Microtask Queue)和事件队列（Event Queue）去实现类似”协程“的异步任务

  • 比较遗憾的是，kotlin协程并不能保证完全的异步，尤其是IO操作和CPU密集型任务

  • Android使用协程进行UI编程的最简单例子

    合理使用协程可以编写出同步风格的代码，也可以避免线程中层层嵌套的”回调地狱“。还可以将协程生命周期和需要的组件的生命周期绑定，如Activity等，避免内存泄露。
